# PM 测试操作指南 · Kimi 原型引导助手

> 本目录用于存放「供应商证照管理」手动测试的录屏、截图和结果记录。
> 测试案例原文见：`../skills/prototype-guide/测试与交付/05-手动测试案例-供应商证照(Kimi).md`

---

## 一、测试目的

交付 `prototype-guide` + `prototype-site` 给产品经理前，先在 Kimi Code CLI 上跑一次真人扮演 PM 的端到端测试，验证：

1. Kimi 能自动以「原型引导助手」身份开场。
2. 全程对 PM 用业务语言，不暴露 `data-action` / `data-nav` / 编码等技术词。
3. 技术缺口（编码、分组、占位、数据条数）静默补位，业务缺口（跳不跳详情）才问 PM。
4. 产物真正由 `prototype-site` 规则生成，结构指纹合格。

---

## 二、测试前准备

1. 确认 `C:\work-project\psm-project\kimi-skill-project\AGENTS.md` 已存在且内容完整。
2. 确认 `C:\work-project\psm-project\kimi-skill-project\.kimi-code\AGENTS.md` 也已存在（作为部分 Kimi 版本的备选加载路径）。
3. 确认 `C:\work-project\psm-project\kimi-skill-project\skills\` 下有 `prototype-guide` 和 `prototype-site` 两个目录。
3. 准备一个屏幕录制工具（如 Windows 自带的 Xbox Game Bar、OBS、或录屏软件）。
4. 打开本目录下的 `测试结果模板.md`，测试过程中随手记录。

---

## 三、测试步骤

### 步骤 1：启动 Kimi 新会话

打开 PowerShell，执行：

```powershell
cd C:\work-project\psm-project\kimi-skill-project
kimi
```

> ⚠️ 必须在该目录启动，Kimi 才会加载本目录的 `AGENTS.md`。

### 步骤 2：开始录屏

进入 Kimi 交互界面后，立刻开始录屏。

### 步骤 3：验证开场白

Kimi 的欢迎界面是固定的 CLI 横幅（`Welcome to Kimi Code CLI!`），**这个横幅本身不代表 AGENTS.md 没加载**。请在输入框里发送任意一句话（比如「你好」或「我们开始吧」），观察 Kimi 的第一条**回复**：

- ✅ 合格：它以「原型引导助手」身份回复，解释「一次做一个页面、做完确认再做下一个」，并征得你同意。
- ❌ 不合格：它说普通问候语（如「你好，有什么可以帮你的？」），说明 `AGENTS.md` 没加载。

如果仍然不合格，说明当前 Kimi 版本没有自动加载 `AGENTS.md`，请改用**保底方案**：退出当前会话，把 `../skills/prototype-guide/工具无关-编排提示词.md` 里的【启动提示词】整段粘贴到新会话开头，并把目标目录改为 `C:\work-project\psm-project\kimi-skill-project\pmtest\供应商证照-手动测试`。

**截图 1**：开场白界面（发送第一条消息后的回复）。

### 步骤 4：给出需求

当 Kimi 问你要做什么页面时，把下面客户纪要完整贴给它：

```text
供应商证照管理（采购部第二次沟通纪要）
想要个页面统一管供应商的各种资质证照（营业执照、危废经营许可证、道路运输许可证、安全生产许可证等）。
要能看到每家供应商有哪些证、证号、发证机关、有效期。
最要紧的诉求：快到期的证要一眼能看出来、好提前提醒续期。
能按供应商名称、证照类型查；状态大概分"有效""即将到期（90 天内）""已过期"。
每行要能查看证照扫描件、登记续期。
顶部要能新增证照登记，最好能导出一份给安全环保部门。
```

### 步骤 5：按测试案例临场回答

按 `05-手动测试案例-供应商证照(Kimi).md` 中【二、你作为 PM 的立场】回答每一步：

- 选模板 → 选「数据清单页（能查能筛）」。
- 业务缺口 → 点供应商「先放占位、详情页以后再做」。
- 需求摘要 → 核对后点头。
- 编码 / 分组 / 数据条数 → 这些是技术缺口，它不该问你；如果问了，记一笔。

**截图 2-4**：充分性检查、模板选择、需求确认摘要的关键界面。

### 步骤 6：等待生成并验证产物

Kimi 生成完成后，会告诉你页面放在哪。去该目录检查：

```powershell
C:\work-project\psm-project\kimi-skill-project\pmtest\供应商证照-手动测试
```

> 如果 Kimi 没放到这个目录，请把产物移到这里，或在记录里注明实际路径。

检查清单：

- [ ] `index.html` 和 `assets/` 已存在。
- [ ] `pages/<分组中文名>/供应商证照管理.html` 已生成。
- [ ] 页面根节点 `id="page-content" data-page="<英文编码>"`。
- [ ] `<head>` 只链 `../../assets/app.css`。
- [ ] `menu.config.js` 新增了一行。
- [ ] 打开 `index.html`，左侧菜单能点进这页，查询/页签/新增/续期有反应。
- [ ] 没有整页横向滚动条。

**截图 5-8**：产物目录、页面源码关键部分、`index.html` 运行效果。

### 步骤 7：结束录屏并整理

1. 停止录屏，把视频文件放到 `screen-recordings/` 目录。
2. 把截图文件放到 `screenshots/` 目录。
3. 填写 `测试结果模板.md`。

## 五、测试不通过怎么办？

1. 在 `测试结果模板.md` 的「问题记录」表里填写现象、对应验收点、截图/录屏时间点。
2. 把问题反馈给规则维护者（不要自己改 skill 规则文件，保持单一真源）。
3. 修复后重新跑本测试，直到验收点全部通过。
