# prototype-guide 手动测试 & 演示方案（Claude 版）

> 用途：你照着这份脚本在 Claude Code 里手动跑一遍 `prototype-guide`，**全程录屏**，给团队演示"产品经理零门槛做原型"。
> 配套案例见同目录 `02-统一测试案例(危废承运商).md`（Claude / Kimi 共用同一案例，方便横向对比）。

---

## 一、开始前准备

1. **确认 skill 能被识别**：用你现在调用 `prototype-site` / 原型制作 的同样方式，确认 Claude Code 能发现 `skills/prototype-guide`。若识别不到，把它放进你环境里加载 skill 的目录后重启。
2. **准备一个空的输出目录**：例如 `D:\演示\承运商原型`，作为生成原型站点的目标目录。
3. **准备案例输入**：把 `02-统一测试案例` 里的「客户沟通纪要」文本复制出来备用（演示时粘贴给 Claude）。
4. 录屏软件就绪，建议同时录下「你输入的话」和「Claude 的回应」。

---

## 二、演示主线（正向流程，约 3~5 分钟）

> 目标：展示 PM 只说大白话，AI 全程引导、自动补技术细节、最后生成可看的原型页。

| 步骤 | 你（扮演产品经理）做什么 | 期望看到 Claude 怎么做（演示卖点） |
|---|---|---|
| 1 | 输入：**「用 prototype-guide 帮我做个原型，目标目录 D:\演示\承运商原型」** | 自我介绍 + **主动讲"逐页推进"工作方式**并征得同意（卖点：贴心、立规矩） |
| 2 | 回复「好」，然后**粘贴案例里的客户沟通纪要** | 给出 ✅我读到了 / ❓没读到 两个清单（卖点：把"读懂没读懂"摆明） |
| 3 | 不用管技术问题，等它建议 | **自动建议「数据清单页」模板**，用业务语言确认（卖点：不让 PM 选编号、不提技术词） |
| 4 | 它问"详情跳哪"——回复「**先放占位，以后再说**」(选 A) | 把缺口收敛成二选一，按你选的 A 处理（卖点：永不卡死、不替你拍板） |
| 5 | 它给"需求确认摘要"——回复「**对**」 | 用大白话复述将生成什么，**全程不出现 data-action 等技术词**（卖点：PM 看得懂） |
| 6 | 等待生成 | 生成原型页 + 注册菜单（卖点：一键出页面） |
| 7 | 看它的收尾 | 给出**补位清单**（编码/分组/跳转占位是它替你补的）+ 问"要不要做下一页"（卖点：透明 + 可持续） |

**演示完打开成果**：在目标目录起本地服务（或让 Claude 起），打开 `index.html`，点左侧菜单进入"危废处置承运商管理"，展示：能查询、能切状态页签、能点新增弹窗、点名称有跳转意图——**原型是真能点的**。

---

## 三、加分演示（反向流程，展示兜底，约 1 分钟）

> 目标：展示"输入很烂时 AI 不崩、不乱来"。

- 另起一次，输入一句很模糊的：**「我要做个管供应商的页面」**（什么都不给）。
- 期望 Claude：不拒绝、不硬编一个错的，而是**先问最关键的 1~2 个业务问题**（这页要查什么 / 显示哪几列），或给"我先按最常见的清单页做一版，你再改"的二选一。
- 这段能让团队看到：**再随便的输入也有合理结果**。

---

## 四、录屏要点（剪辑/讲解时强调这几个卖点）

1. PM **全程只说业务大白话**，没写一个技术标记。
2. AI **主动引导 + 报进度**，不是干等指令。
3. **一次只做一页**、做完确认（讲清"慢就是快"）。
4. 信息不够时**给选择题**，而不是拒绝或乱做。
5. 最后**老实交代补了什么**（补位清单）。
6. 产出是**真能点的原型**，可直接拿去跟客户聊。

---

## 五、验收清单（边录边勾）

- [ ] 开场解释了"逐页推进"并征得同意
- [ ] 对粘贴的纪要做了"读到/没读到"两清单
- [ ] 自动建议了模板并用业务名确认（没让你选编号、没提技术词）
- [ ] 缺口处理是"提醒 + 二选一"，没把技术问题（编码/分组）丢给你
- [ ] 给了业务语言"需求确认摘要"
- [ ] 真生成了页面 + 注册了菜单
- [ ] 给了补位清单 + 问是否继续下一页
- [ ] 打开站点：页面能进、能查、能切页签、能弹窗、跳转有意图

---

## 六、出问题怎么记录（反馈给开发）

每发现一个问题，记一条：
- **在第几步**、**你说了什么**、**AI 实际怎么做的**、**你期望它怎么做**。
- 截图或录屏时间点。
这样我能精确定位是 flow / 模板 / 还是 prototype-site 生成层的问题。
