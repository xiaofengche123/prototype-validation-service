# 模板 · 内嵌弹窗/抽屉（非独立页面）

> 内部页型：内嵌弹窗 / 内嵌抽屉。用于在独立页面中弹出的表单、确认、详情等，**不注册菜单、不生成独立 .html 文件**。
> 三段结构：① PM 可见的业务要素（用元追问收集）② 内置约束（隐藏，含 IR）③ 元追问清单。

---

## ① 业务要素（要从 PM 那里拿到的，全用业务语言）

- 弹窗/抽屉叫什么（中文名）、由哪个页面触发（如：任务验收审核页点击"业务核实"按钮）
- 弹窗/抽屉里要**显示/填写哪些内容**（如：核实意见、附件、确认按钮）
- 有哪些**操作按钮**（如：确认通过 / 退回修改 / 取消）
- 点击按钮后要**做什么**（如：关闭弹窗并刷新父页列表、跳转到另一个页面）
- 弹窗/抽屉的**宽度**（默认 720px 标准弹窗；宽表单用 960px；侧边抽屉用 480px）

> 凡 PM 没说的，按"内置约束 + 业务常识"补位并标【补位】，不要追着 PM 一条条逼问。

---

## ② 内置约束（PM 不可见 · 生成时必须落实）

生成遵循 `../../prototype-site/SKILL.md` 与 `../../prototype-site/references/interaction-ir.md`、`visual-parity.md`。本页型固定产出：

### 核心规则

- **不生成独立 .html 文件**：产物写入**触发父页面**的 HTML 中，作为 `div` 弹窗/抽屉容器
- **不注册 menu.config.js**：内嵌组件无菜单项
- **页面编码**：使用 `【页型：内嵌弹窗】` 标记中的编码，但仅作为 `data-modal` 的标识，不用于菜单注册

### 弹窗结构（标准 720px）

```html
<!-- 弹窗容器：放在触发页面的业务内容区末尾，不在 head 或 body 外 -->
<div class="modal-mask" id="verifyModalMask" data-modal="verifyModal">
  <div class="modal" style="width:720px">
    <div class="modal-title">业务核实确认</div>
    <div class="modal-body">
      <!-- 表单按 2 列网格组织 -->
      <div class="prototype-modal-form">
        <div class="prototype-field">
          <label>核实意见</label>
          <textarea data-field="verifyOpinion"></textarea>
        </div>
        <!-- ... 其他字段 -->
      </div>
    </div>
    <div class="modal-footer">
      <button data-action="close-modal" data-modal-target="verifyModal">取消</button>
      <button class="btn-primary" data-action="save-verify" data-modal-target="verifyModal">确认通过</button>
    </div>
  </div>
</div>
```

### 抽屉结构（右侧滑出，480px）

```html
<!-- 抽屉容器：fixed 定位，右侧滑出 -->
<div class="drawer-mask" id="archiveDrawerMask" data-modal="archiveDrawer">
  <div class="drawer" style="width:480px">
    <div class="drawer-header">
      <span class="drawer-title">归档入库</span>
      <button class="drawer-close" data-action="close-modal" data-modal-target="archiveDrawer">×</button>
    </div>
    <div class="drawer-body">
      <!-- 内容区 -->
    </div>
    <div class="drawer-footer">
      <button data-action="close-modal" data-modal-target="archiveDrawer">取消</button>
      <button class="btn-primary" data-action="save-archive" data-modal-target="archiveDrawer">确认归档</button>
    </div>
  </div>
</div>
```

### 触发按钮（在父页面中）

```html
<button data-action="open-verify" data-modal-target="verifyModal">业务核实</button>
```

### IR 标记规范

- 触发按钮：`data-action="open-<编码>"` + `data-modal-target="<弹窗编码>"`
- 弹窗容器：`data-modal="<弹窗编码>"`
- 关闭按钮：`data-action="close-modal"` + `data-modal-target="<弹窗编码>"`
- 保存/提交按钮：`data-action="save-<动作>"` + `data-modal-target="<弹窗编码>"`

### 样式约束

- 弹窗/抽屉样式写入 `assets/theme.css` 或复用 `app.css` 中的 `.modal-mask` / `.drawer-mask` 类
- **不允许在父页面中写 `<style>` 块或内联 `style="..."`**（B 路线除外：自定义样式写入 `assets/theme.css`）
- 弹窗宽度：标准 720px、宽表单 960px、抽屉 480px
- 表单布局：2 列网格（`.prototype-modal-form`），与独立页面的弹窗一致

### 与父页面的关系

- 内嵌组件的编码必须在父页面的 `data-page` 上下文中唯一
- 同一父页面可有多个内嵌组件，编码不可重复
- 内嵌组件的 mock 数据与父页面共享数据上下文（如：父页面列表选中行的数据传入弹窗）

## ④ 与独立页面的区别（AI 必须区分）

| 维度 | 独立页面 | 内嵌弹窗/抽屉 |
|------|---------|--------------|
| 产物文件 | `pages/<分组>/<页面名>.html` | 嵌入父页面 HTML 中 |
| 菜单注册 | ✅ 幂等注册 `menu.config.js` | ❌ 不注册 |
| `data-page` | 根节点 `id="page-content"` | 无，使用 `data-modal` |
| shell 标记 | ❌ 无 | ❌ 无（在父页面内） |
| 独立访问 | 可通过菜单/URL 打开 | 只能通过父页面触发 |
| 编码用途 | 菜单编码 + 跳转目标 | 仅弹窗标识 |

## ⑤ 常见错误（必须避免）

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 把内嵌组件生成独立 .html | 菜单多出一项，点击打开空白页 | 嵌入父页面，不注册菜单 |
| 内嵌组件写 `<head>` / `<body>` | 产物结构错误，转 Vue 失败 | 只写弹窗容器 div |
| 内嵌组件编码与父页面重复 | 跳转/弹窗冲突 | 编码在父页面上下文中唯一 |
| 内嵌组件写内联 `<style>` | 违反规则，样式无法复用 | 样式写入 `assets/theme.css` |
| 抽屉用 `position:sticky` | 滚动时抖动 | 用 `fixed` 定位 + 遮罩层 |
