# 模板 · 参数配置页

> 内部页型：配置矩阵页。配一张可编辑的参数矩阵、带自动计算和试算时用。
> 三段结构：① PM 可见业务要素 ② 内置约束（隐藏，含 IR）③ 元追问清单。

---

## ① 业务要素（业务语言）

- 页面中文名、菜单分组
- 顶部说明条：解释这套参数配什么、怎么算
- 配置矩阵：**行是什么维度**（元素/指标/工序…）、**列是什么维度**（工序1/2/3/综合结果…）；哪些格子可编辑、哪些列是自动算出来的
- 右侧说明卡：点某行时联动显示该行的业务说明/公式/建议
- 试算卡：输入业务参数 → 预览计算结果
- 底部操作：重置默认、预览传导、保存

## ② 内置约束（PM 不可见 · 生成时落实）

遵循 `../../prototype-site/SKILL.md` 与 `interaction-ir.md`：

- 矩阵卡：可编辑单元格用输入框；自动计算列**只读 + 蓝色高亮**；改输入框后自动刷新综合结果。
- 点矩阵某行 → 右侧说明卡联动更新。
- 试算 `data-action="preview"`；底部 `reset-default` / `preview` / `save`；保存前做**范围校验**。
- **底部操作栏固定**（`has-fixed-actions` + `.fixed-bottom-bar`）。
- "滚动/定位某块"行为（如预览后滚到结果卡）：原型用 DOM，但**为后续转 Vue，定位应基于具名元素而非全局 DOM 查询**（转 Vue 时走 `ref`，避免 `$el.querySelector`）。
- 矩阵过宽**只在卡内横滚**，不得页面级横滚；左右卡等高。
- 菜单幂等注册。

## ③ 元追问清单（业务语言，少量多次）

1. 这套参数配的是什么？页面叫什么、放哪个分组？
2. 矩阵的**行**代表什么？**列**代表什么？
3. 哪些格子人来填、哪些列**系统自动算**？
4. 点某一行要不要在旁边显示**说明或公式**？
5. 要不要"试算/预览"看一下效果？
6. 保存时有没有**取值范围**要校验？

> 行列维度是关键，必须问到；公式/范围缺则标【补位】。
