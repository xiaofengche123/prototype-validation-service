# 原型制作 skill（站点版）— 使用手册

> 给使用者看的操作手册。重点是**大量可拷贝的提示词案例**——复制下来，改几处必填项就能用。

---

## 1. 这个 skill 帮你做什么

一句话描述业务，它把一张**统一风格、能点能演示、且已为转 Vue 做好准备**的原型页面挂进**一个共用外壳的站点**（默认 `xdocs/原型站点/`）。

特点：
- **整站可演示**：顶栏 + 左菜单 + 页签全站共用，新页面自动挂菜单，可整目录打包给客户做**多页联动**演示（点单号跳明细、跨页跳转都真能走）。
- **风格永远统一**：风格只由 `assets/app.css` 决定，业务页不带样式，永远是绿星链通主色 `#1565c0`。
- **画页面更专注**：每张业务页文件里**只有业务内容**，大模型不被外壳布局干扰。
- **交互自动补位**：你没写清的按钮也"点了有反应"，并为转 Vue 埋好标记。
- **转 Vue 友好**：业务页已与外壳分离，可被「原型转Vue」skill 直接、高质量地转成项目页面，无需剥壳。

---

## 2. 怎么用（最简）

在对话里调用 skill，给一段业务描述即可。第一次用会自动在站点目录铺好框架（`index.html`/`assets`/`menu.config.js`/`pages`），之后每次只产出一张 `pages/<分组中文名>/<页面中文名>.html`（文件夹按菜单分组、文件用中文名）并挂上菜单。

**预览两种方式：**
- 整站：浏览器打开站点根 `index.html`（双击即可，无需起服务）→ 左菜单点你的页面，体验多页签 + 跨页联动。
- 单页：直接打开 `pages/<分组中文名>/<页面中文名>.html` 快速看本页。

### 推荐你提供的要素（给得越多越好，但都可省略）
- **页面编码**（如 `BMSP01`）：模块 BM + 子模块 SP + 序号 01。不给则它建议一个。
- **页面名称**（如 供应商准入申请）。
- **查询字段**：字段名 + 类型（输入框/下拉/日期范围）+ 是否必填。
- **表格列**：列名，哪列是可点链接（跳哪个编码），哪列是状态标签。
- **状态集合**：如 草稿/待审核/已通过/已驳回。
- **行操作**：不同状态下显示哪些按钮（编辑/删除/查看/审核…）。
- **顶部操作**：新增/批量导出/批量审批…
- **菜单归属**：放哪个分组（基础信息管理/计划管理/合同管理…），不给则按业务归类。

## 3. 提示词案例库（复制即用）

> 用法：复制一个案例 → 把【】里的内容换成你的 → 发给 skill。

### 案例 A：列表查询页（信息最全的"黄金范例"）

```
用「原型制作」skill 做一个原型页面，加到 xdocs/原型站点/。

页面编码：BMSP01
页面名称：供应商准入申请
菜单分组：基础信息管理
页面类型：列表查询页

查询字段：
- 申请单号：输入框
- 供应商名称：输入框，必填
- 申请类型：下拉（新增准入/信息变更/续签延期）
- 申请状态：下拉（全部/草稿/待审核/已通过/已驳回）
- 提交日期：日期范围

统计卡片：全部/草稿/待审核/已通过/已驳回

表格列：序号、申请单号(可点，跳明细 BMSP01-DETAIL)、供应商名称、统一社会信用代码、
        申请类型、申请状态(彩色标签)、提交人、提交时间、操作

行操作（按状态）：
- 草稿：编辑(弹窗)、删除
- 待审核：查看、审核、撤回
- 已通过：查看、打印
- 已驳回：查看、重新提交

顶部操作：新增申请(弹窗)、批量导出、批量审批
弹窗：新增/编辑表单，字段=供应商名称、统一社会信用代码、申请类型、提交人
样例数据 20 行，覆盖全部状态。
```

### 案例 B：极简输入（考验自动补位）

```
用「原型制作」skill 做一个"物料主数据管理"列表查询页，编码 BMWL01，挂到基础信息管理分组。
```
> skill 会自动补常见字段/列/状态/增删改查行操作/新增弹窗，并在规格卡里标 `【补位】` 供你确认；同时挂上菜单。

### 案例 C：表单录入页

```
用「原型制作」skill 做一个"供应商准入申请填报"表单页，编码 BMSP02，挂基础信息管理。
分组录入：
- 基本信息：供应商名称(必填)、统一社会信用代码(必填)、所属行业(下拉)、成立日期
- 联系信息：联系人、电话、邮箱、地址
- 资质信息：营业执照(上传)、资质等级(下拉)、有效期
底部操作：保存草稿、提交审核、返回(跳 BMSP01)
```

### 案例 D：详情/查看页（演示从列表跳进来）

```
用「原型制作」skill 做一个"供应商准入申请明细"详情页，编码 BMSP01-DETAIL。
（列表 BMSP01 单号已 data-nav 到这里，做出来即可联动）
顶部：申请单号、当前状态标签、基本信息只读展示。
中部：标签页分「基本信息 / 资质附件 / 审批记录」，审批记录用时间轴。
底部：返回(跳 BMSP01)、打印。
```

### 案例 E：在已有原型上迭代

```
用「原型制作」skill 修改 xdocs/原型站点/pages/基础信息管理/供应商准入申请.html：
1. 查询区增加"准入类别"下拉
2. 表格增加"当前审批节点"列，放在"提交人"前
3. 草稿状态增加"提交审核"按钮
4. 样例数据加到 30 行
菜单不用动。
```

### 案例 F：搭一套连贯演示（多页联动）

```
用「原型制作」skill 在 xdocs/原型站点/ 下连做三页，互相能跳：
1. BMSP01 供应商准入申请（列表，单号跳 BMSP01-DETAIL，新增跳 BMSP02）
2. BMSP02 准入申请填报（表单，返回跳 BMSP01）
3. BMSP01-DETAIL 准入申请明细（详情，返回跳 BMSP01）
都挂到基础信息管理分组。
```

## 4. 产出长什么样

- 站点目录 `xdocs/原型站点/`：`index.html` + `assets/` + `menu.config.js` + `pages/<分组中文名>/<页面中文名>.html`。
- 打开 `index.html` 就是一个完整的"管理系统"演示：顶栏、左菜单、多页签、内容区切换。
- 每个 `pages/*.html` 顶部有**规格卡**注释（编码/字段/状态/操作去向）——后续转 Vue 的依据。
- 所有按钮可点、弹窗可开关、跳转有反应。
