# erp-prototype-site 使用手册（对外分享版）

本手册用于指导同事使用 `erp-prototype-site` skill 生成 PSM 风格静态原型页面。

## 1. 先说结论

新版 `erp-prototype-site` skill 的设计目标是：

- 第一次生成一个共用外壳站点：顶部栏、左侧菜单、页签栏、公共样式、运行脚本。
- 后续每次只生成或修改一个业务页面。
- 业务页面只写主内容区，不重复顶部栏、侧边栏、页签栏。
- 后续转 Vue 时只取业务页的 `#page-content` 和弹窗内容，避免框架噪声影响转换。

因此，使用者写提示词时只需要描述主内容区。

## 2. 使用时不要写什么

不要在提示词里详细描述这些框架级内容：

- 顶部蓝色 header
- 左侧菜单完整结构
- 页面标签栏、多标签关闭逻辑
- 系统 logo、公司名称、用户头像
- iframe、整体 shell 布局

这些由 skill 的共享外壳统一提供。

如果需要把页面挂到菜单里，只写：

```text
菜单分组：工艺参数配置
```

不要写完整菜单树。

## 3. 推荐提示词结构

每次生成页面，建议按下面结构写：

```text
用 erp-prototype-site skill 在【目标目录】生成一个原型页面。

页面编码：【页面编码】
页面名称：【页面中文名】
菜单分组：【菜单分组中文名】
页面类型：【列表查询页 / 表单页 / 详情页 / 看板页 / 配置页 / 对比分析页 / 审批页】

只生成主内容区，不要生成顶部栏、左侧菜单、页签栏等框架内容。

业务说明：
【描述页面用途和业务背景】

页面结构：
【写查询区、列表区、表单区、卡片区、弹窗、底部操作栏等】

交互要求：
【写 data-action / data-nav / data-modal 对应行为】

MOCK 数据：
【写样例数据，或说明由 skill 补位】
```

## 4. 通用占位说明

复制模板后，只需要修改这些位置：

- `【目标目录】`：原型站点保存目录，例如 `D:\项目\第二轮测试`
- `【页面编码】`：例如 `BMGCS01`
- `【页面中文名】`：例如 `工艺参数总览`
- `【菜单分组中文名】`：例如 `工艺参数配置`
- `【页面类型】`：按实际页面选择
- 业务字段、表格列、按钮、弹窗、MOCK 数据

### 模板 1：最常用列表查询页

```text
用 erp-prototype-site skill 在【目标目录】生成一个原型页面。

页面编码：【页面编码】
页面名称：【页面中文名】
菜单分组：【菜单分组中文名】
页面类型：列表查询页

只生成主内容区，不要生成顶部栏、左侧菜单、页签栏等框架内容。

查询区字段：
- 【字段1】：输入框，placeholder：【提示文字】
- 【字段2】：下拉框，选项：【选项1 / 选项2 / 选项3】
- 【字段3】：日期范围，默认：【开始日期】 至 【结束日期】
- 【字段4】：下拉框，选项：【全部 / 有效 / 失效】

列表区：
- 工具栏：标题【列表标题】、总数角标、按钮【新增 / 批量导出 / 列表设置】
- 状态筛选标签：【全部 / 有效 / 失效】，带数量角标
- 表格列：【列1】、【列2】、【列3】、【状态列】、【最近维护时间】、【操作】
- 状态列用彩色标签显示
- 操作列包含：【查看】、【编辑】、【历史】

交互要求：
- 查询按钮 data-action="search"，点击后显示短暂骨架屏或加载态
- 重置按钮 data-action="reset"，恢复默认条件
- 新增按钮 data-action="open-add"，打开新增弹窗或提示桩
- 编辑按钮 data-action="open-edit"
- 历史按钮 data-action="open-history"，打开历史版本弹窗
- 表格行选择后显示批量操作栏
- 分页支持首页、上一页、下一页、末页、页大小选择、跳转

MOCK 数据：
请生成【数量】条数据，覆盖全部状态。
```

### 模板 13：基于旧 HTML 页面生成提示词（再走原有流程）

```text
用 erp-prototype-site skill 在【目标目录】基于旧 HTML 页面生成新原型页面。

旧页面来源：【旧 HTML 文件路径，或粘贴 HTML 代码】

处理流程：
1. 剥离旧页面的外壳（顶栏/侧边栏/页签），提取业务内容
2. 从业务内容中提取：查询字段、表格列、操作按钮、弹窗内容、状态分类
3. 将提取结果写成提示词文件 prompts/<编码>-<页面名>.md
4. 按提示词走原有流程：充分性检查 → 模板建议 → 缺口处理 → 确认 → 生成

基础信息（从旧页面提取或推断）：
- 页面编码：【新页面编码，或让 skill 自动分配（6 位合规）】
- 页面名称：【从旧页面 <title> 或 <h1> 提取】
- 菜单分组：【从旧页面路径推断，或用户指定】
- 页面类型：【推断的页面类型：列表查询页/填表录入页/详情查看页/...】

风格选择（门一豁免，改为风格确认）：
- A：按高端ERP默认风格重新生成（推荐）
- B：提取旧页面颜色/字体/边框写入 theme.css，尽量接近原风格
- C：最小改动，只注入 IR 标记，保留原结构

提取缺口（AI 无法识别或不确定的部分，必填）：
- 【补位】旧页面中【某按钮/某字段/某交互】未识别，按常见做法处理为...
- 【补位】...
- （无缺口写"无"）
```

> 注意：本模板的产物是**提示词文件**，不是直接生成页面。提示词文件写入后，走和文档/口述输入完全相同的后续流程（充分性检查 → 模板建议 → 缺口处理 → 确认 → 生成）。
