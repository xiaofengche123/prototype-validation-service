# 模板向导 skill（template-wizard）— 使用手册

> 这个 skill 存在的目的就是让你**不用写提示词**。所以这份手册特别短：你只需要会说下面几句话。

---

## 1. 它帮你做什么

「模板提炼」和「模板校准」是两台引擎，功能全但启动要写结构化提示词。向导是驾驶舱：

- 你说一句大白话，它自动检测现在进行到哪一步（查风格包版本、查校准台账）；
- 它用**一次一个的选择题**问你，每个问题带推荐答案和一句"为什么问"；
- 提示词、契约、台账全部由它替你填，引擎由它替你调；
- 你只做两件事：**回答问题**和**在确认门看页面点头/摇头**。

它不会跳过任何引擎纪律（确认门、问题三分类、回灌 7 步都在），只是把它们翻译成人话。

---

## 2. 你只需要会说这几句话

| 场景 | 你说 |
| --- | --- |
| 第一次做新风格 | `用「模板向导」开始做新模板` |
| 隔了几天回来 | `用「模板向导」继续` |
| 不知道现在啥状态 | `用「模板向导」看下现在到哪一步了` |
| 发现原型和 Vue 长得不一样 | `用「模板向导」，Vue 页面和原型不一致了，帮我查` |
| 中途想撤 | `先停，把现场记好` |

就这些。剩下的它会问你。

## 3. 一次完整对话长什么样（示例，让你有预期）

```text
你：用「模板向导」开始做新模板

向导：查了一下：style-packs 里没有新包、也没有进行中的校准，这是全新开工。
      先问第一个问题——这次的新样式，你手里有什么？
      ① 能跑起来的旧系统  ② UI设计图（可能带文字说明/HTML）  ③ 现成的静态原型页
      （选一个或几个就行）

你：②，设计图在 xdocs/资料/新版设计/

向导：收到。给这套风格起个英文代号？我建议 cnal-green（看设计图主色是绿）
      或 cnal-fresh。也可以你自己起。

你：cnal-green

向导：好。提醒一句规矩：那份 demo.html 我只参考"页面上有什么"，颜色间距
      一律以设计图为准。我现在的计划是：
      1. 把设计资产归档登记；
      2. 先照设计图做一张黄金样板页给你看（不先写契约，这是规矩）；
      3. 你确认后我再反向提炼契约。
      没问题我就开工。

你：开工

向导：（干活…）黄金页做好了：style-packs/cnal-green/prototype/golden/xxx.html，
      双击打开。我念一条你看一条：
      1. 主色和状态标签颜色，和设计图对得上吗？
      2. 卡片标题栏高度样式一致吗？
      …

你：第3条不对，查询区控件比设计图矮

向导：记下了：控件高度返工（设计图量出来36px，我做成了32px）。改契约和页面，
      几分钟后你再看第二轮。
```

校准阶段同理：批次执行你只看它报的三分类表格打勾，批次末它跑 test/build 给你签结论。

## 4. 常见问题

**Q：向导和直接用两台引擎，产出有区别吗？**
没有。向导只是替你写提示词和调引擎，产物（风格包、台账、家族）完全一样。哪天你熟了想直接用引擎的案例库，随时可以，两边不冲突。

**Q：它会不会替我做决定？**
不会。所有确认门、问题裁决、批次结论都停下来等你。它只给推荐和理由。

**Q：会话断了/换电脑了怎么办？**
说"用「模板向导」继续"。它靠文件（接手说明、台账、向导进度）恢复现场，不靠聊天记录。

**Q：我只是想做一张原型页，不是做模板？**
直接用「原型制作」skill 就行，向导会主动告诉你别走这条流程。

**Q：向导问的问题我答不上来？**
说"不知道/你推荐"。每个问题都有默认答案，它会用推荐值继续并记下这是默认决策，之后可以改。
