# 统一测试案例 · 危废处置承运商管理

> 这是**所有工具（Claude / Kimi / Codex / Trae）共用的同一个测试案例**，目的是让不同工具跑同一输入，横向对比表现。
> 用法：把下面【客户沟通纪要】整段作为输入贴给被测工具的 erp-prototype-guide；按【标准交互答案】扮演 PM 回应；用【期望产物】和【验收点】对照结果。

---

## 一、输入：客户沟通纪要（原样贴给工具）

```
危废承运商管理（第二次客户沟通节选）
客户想要个页面管他们的"危废处置承运商"。要能看到所有承运商，能按名称搜、按资质类型筛。
列表显示：承运商名称、资质类型、联系人、电话、合作状态。
客户说有的承运商"合作中"、有的"已暂停"。每行要能查看详情和编辑。
顶部要能新增承运商，最好能导出。
（没提：菜单放哪、点详情跳哪页、要多少数据）
```

## 二、标准交互答案（你扮演 PM 时这样回）

| 工具问你 | 你的标准回答 |
|---|---|
| 是否同意"逐页推进"工作方式 | 同意 |
| 给需求 | 粘贴上面的纪要 |
| 建议「数据清单页」模板，确认吗 | 确认 |
| 详情跳到哪页（业务性缺口，二选一） | 选 A：先放占位、以后再做 |
| 需求确认摘要对不对 | 对 |
| 是否继续下一页 | 否（本次演示到此） |

> 关键观察：工具**应该自动补**编码/菜单分组/跳转占位/数据条数（技术性缺口），**不应该**反过来问你这些。

## 三、期望产物

- 一个原型站点（若目标目录为空则先建共享框架），其中：
  - 业务页：`pages/基础信息管理/危废处置承运商管理.html`，根节点 `data-page="BMCY01"`（6 位编码：BM 模块 + CY 子模块 + 01 序号）
  - 菜单：`menu.config.js` 的"基础信息管理"分组下新增一行 `BMCY01`
- 页面包含：
  - 查询区：承运商名称(输入) / 资质类型(下拉) / 合作状态(下拉)
  - 状态：全部 / 合作中 / 已暂停（状态页签 + 统计卡）
  - 表格列：序号 / 承运商名称(可点跳详情) / 资质类型 / 联系人 / 联系电话 / 合作状态(彩色标签) / 操作(查看+编辑)
  - 顶部：新增承运商(弹窗) / 批量导出
  - 弹窗：新增/编辑承运商表单
  - 示例数据 约 10 条，覆盖合作中/已暂停

> 参考标准产物：仓库里 `vue3-project-skill/guide-test-site/`（Claude 跑出来的版本），可作为"长什么样才算对"的对照。

## 四、验收点（每个工具都按这套打分）

1. 全程**没向 PM 暴露**任何技术标记（data-action / data-nav / data-modal / 编码 等）。
2. 自动**选对模板**（数据清单页）。
3. **技术性缺口静默补位**（编码 `BMCY01`（按 menu.config.js 顺延，6 位 2+2+2 结构）、分组"基础信息管理"、跳转占位 `BMCY02`（占位详情页，未注册 menu）、数据 10 条），并在规格卡补位项栏逐条记录，交付时从规格卡读出补位清单。
4. **业务性缺口才问**（详情跳哪），且是"提醒 + 二选一"。
5. 产物结构符合"期望产物"，页面**真能点**（查询/切页签/弹窗/跳转意图）。
6. 没有页面级横向滚动；窄表不强制横滚；有底部按钮则固定底栏（本案例无底部栏）。
