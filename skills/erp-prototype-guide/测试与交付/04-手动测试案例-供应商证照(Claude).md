## 一、客户纪要（等 Claude 问你要需求时，把这段贴给它）

```
供应商证照管理（采购部第二次沟通纪要）
想要个页面统一管供应商的各种资质证照（营业执照、危废经营许可证、道路运输许可证、安全生产许可证等）。
要能看到每家供应商有哪些证、证号、发证机关、有效期。
最要紧的诉求：快到期的证要一眼能看出来、好提前提醒续期。
能按供应商名称、证照类型查；状态大概分"有效""即将到期（90 天内）""已过期"。
每行要能查看证照扫描件、登记续期。
顶部要能新增证照登记，最好能导出一份给安全环保部门。
```

> 故意没说：放哪个菜单、点供应商要不要跳详情页、要展示多少条数据。
> （这是在试 Claude 会不会"技术缺口自己补、业务缺口才问你"。）

## 二、你作为 PM 的立场（Claude 问到时你这样临场决定——这是你的权，别让它替你拍）

- 它让你**选模板** → 你选「数据清单页（能查能筛）」。
  （观察：它有没有给你业务化清单让你选，还是自作主张直接定。）
- 它问**业务缺口**「点供应商要不要跳详情页」 → 你回：先放占位、详情页以后再做。
- 它给**需求确认摘要** → 你核对：能按 供应商名称 / 证照类型 / 状态 查；
  列要有 供应商、证照类型、证号、发证机关、有效期、状态、操作；快到期的要醒目。对就点头。
- 它若反过来问你**编码 / 菜单分组 / 数据条数** → 这些它【本该自己补】；
  它要是拿来烦你，记一笔（不合格信号）。

## 三、验收点（逐条打勾）

- [ ] 全程**没蹦技术词**：没出现 data-action / data-nav / 编码 / IR 等字样。
- [ ] **让你选模板**（给了业务化清单或至少业务名确认），没替你拍板。
- [ ] **没替你编答案**：每个该你定的，它都停下等你回。
- [ ] **技术缺口它自己补了**（编码、菜单分组、详情页跳转占位、数据条数），没拿来烦你。
- [ ] **业务缺口才问你**（跳不跳详情），且是"提醒 + 二选一"。
- [ ] ⭐ **真走进了 erp-prototype-site**——去目标目录看产物：
  - [ ] 页面在 `pages/<分组>/供应商证照管理.html`，根节点有 `data-page`
  - [ ] `<head>` 只链 `app.css`、无内联 `<style>`、无外壳（顶栏/菜单/页签）标记
  - [ ] `menu.config.js` 多了一行
  - [ ] 打开站点 `index.html`，左侧菜单能点开这页；查询 / 状态页签 / 新增弹窗 / 续期都有反应
  - [ ] 没有整页横向滚动条
- [ ] 结尾给了**补位清单**（列出它替你补的编码 / 分组 / 占位 / 条数），并问要不要做下一页。

> 只要 ⭐ 那条过了，就证明 A 方案**焊住了底线**——请求真的落到 erp-prototype-site，
> 而不是被别的方式（比如老的单页 skill）画出来。这正是上一轮翻车的那个点。

## 附：预期它大概会怎么补位（仅供你 review 时对照，别提前告诉它）

- 页面编码：按 `menu.config.js` 现有编码顺延推一个（如 `BMSPZ01` 之类），不问你。编码必须 6 位（2位模块+2位子模块+2位序号）。
- 菜单分组：归到「基础信息管理」之类最贴的现有分组，不问你。
- 详情跳转：你选了"先占位"，它分配合规 6 位编码（如 `BMSP02`）做占位、不注册进 menu、规格卡补位项标明"占位"；点了语义正确（目标页以后做）。
- 数据条数：默认 10 条左右，覆盖 有效 / 即将到期 / 已过期 三种状态。
