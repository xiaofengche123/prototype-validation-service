# 模板 · 数据清单页（能查能筛）

> 内部页型：列表查询页。最常用的页面类型。
> 三段结构：① PM 可见的业务要素（用元追问收集）② 内置约束（隐藏，含 IR）③ 元追问清单。

---

## ① 业务要素（要从 PM 那里拿到的，全用业务语言）

- 页面叫什么（中文名）、属于哪个菜单分组
- 要按哪些**条件查询**（如：名称、分类、状态、日期、价格区间）
- 清单里要**显示哪些列**（如：名称、分类、价格、状态、操作）
- 有哪些**状态分类**（如：生效中 / 待生效 / 已过期）——用于状态筛选页签
- 每行有哪些**操作**（如：查看 / 编辑 / 历史 / 删除）
- 顶部有哪些**整体操作**（如：新增 / 批量导出 / 批量导入）
- 点击某列或按钮要**跳到哪个页面**或**弹出什么**（如：点单号看详情、点新增弹表单）
- 要放多少条**示例数据**（默认 8~15 条，覆盖各状态）

> 凡 PM 没说的，按"内置约束 + 业务常识"补位并标【补位】，不要追着 PM 一条条逼问。

## ② 内置约束（PM 不可见 · 生成时必须落实）

生成遵循 `../../prototype-site/SKILL.md` 与 `../../prototype-site/references/interaction-ir.md`、`visual-parity.md`。本页型固定产出：

- 页面根 `id="page-content"` + `data-page="<英文编码>"`，加 `prototype-page`；只链 `app.css`，无 shell 标记、无 inline style。
- **查询卡**：`prototype-card` + `prototype-card-head`(标题+重置/查询按钮) + `prototype-query-grid`(每字段 `prototype-field` + label)。重置 `data-action="reset"`、查询 `data-action="search"`。
- **状态筛选页签**：按 PM 给的状态分类生成，带数量角标；点击过滤。
- **列表卡**：工具栏(标题 + 总数角标 + 整体操作按钮)。新增 `data-action="open-add"` + `data-modal`；批量类 `data-action="batch-*"`。
- **表格**：数据驱动（数组 + render，渲染后 `PR.bindActions()`）；状态列用彩色 `status-tag`；行操作按状态分支（编辑 `open-edit`+`data-modal`、历史 `open-history`+`data-modal`、删除 `delete-row`、查看 `open-view`）。
- **跳转**用 `data-nav="<目标编码>"`（不是 `location.href`）；目标页未做也保留语义。
- **弹窗**用 `data-modal`，能开能关；编辑表单按 2 列网格组织。
- **布局红线**：无页面级横向滚动；窄表(3~6列)用 fit-table 自然铺满、不强制横滚；宽表只在表格 wrapper 内横滚；同排卡片等高；有底部操作按钮则固定底栏。
- **菜单注册**：幂等更新 `menu.config.js` 一行 `{ code, cname, group, icon, file }`。
- 足量 mock 数据，避免大块空白。

## ③ 元追问清单（按需逐条问 PM，业务语言、一次别问太多）

1. 这个清单页**叫什么名字**？大概放在**哪个菜单分组**下？
2. 客户想**按什么来查**这批数据？（给几个常见条件让他点头，别开放式问）
3. 清单里**最该让客户看到哪几列**？
4. 这些数据有**哪几种状态**？（用来做顶部筛选）
5. 每条记录客户可能要做什么？（查看？编辑？看历史？）
6. 顶部要不要**新增 / 导出**这类整体按钮？
7. 点某一行要不要**跳到详情页**或**弹出编辑框**？
8. 示例数据放多少条够你给客户演示？（默认我放 10 条左右）

> 收集到第 1、3、4 条就已经能出一版了；其余缺的走"补位 + 标【补位】"。
