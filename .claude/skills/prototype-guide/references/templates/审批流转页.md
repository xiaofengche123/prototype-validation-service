# 模板 · 审批流转页

> 内部页型：审批流转页。单据审批（意见 + 通过/退回/转办/加签 + 流程时间线）时用。
> 三段结构：① PM 可见业务要素 ② 内置约束（隐藏，含 IR）③ 元追问清单。

---

## ① 业务要素（业务语言）

- 页面中文名、菜单分组
- 顶部单据信息条：单据号、名称、提交人、提交时间
- 摘要卡：几个核心指标（总成本/总量/供应商数/达标率…）
- 审批区（左右两列）：左=审批意见文本域 + 通过/退回/转办/加签按钮；右=审批流程时间线
- 下方明细区：方案对比表 / 分配明细表 / 达标验证表等
- 底部操作：返回、导出审批单

## ② 内置约束（PM 不可见 · 生成时落实）

遵循 `../../prototype-site/SKILL.md` 与 `interaction-ir.md`：

- 顶部单据信息条 + 摘要指标卡。
- 审批区左右两列**等高对齐**：左侧意见文本域 + 动作按钮（通过 `approve`、退回 `reject`、转办 `transfer`、加签 `add-sign`）；右侧时间线。
- **通过/退回必须先填审批意见**（空则提示）；**转办/加签点击后追加时间线节点**。
- 返回 `data-nav="<返回页编码>"`；导出审批单 `data-action="export-approval"`。
- **底部操作栏固定**（`has-fixed-actions` + `.fixed-bottom-bar`）。
- 明细表数据驱动；无页面级横向滚动（过宽只在 wrapper 内横滚）。
- 菜单幂等注册。

## ③ 元追问清单（业务语言，少量多次）

1. 审批的是什么单据？页面叫什么、放哪个分组？
2. 顶部摘要想让审批人先看到哪几个**关键指标**？
3. 审批有哪些动作（通过/退回是必须；要不要转办、加签）？
4. 下面要展示哪些**明细**给审批人看（方案/分配/达标…）？
5. 返回到哪个页面？要不要导出审批单？

> 审批动作 + 明细内容是关键；指标/口径缺则标【补位】。
