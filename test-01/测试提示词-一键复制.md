# 05 方案验证 · 测试提示词（一键复制）

> **用法**：一个用例 = 一个新会话。复制对应提示词 → 粘贴 → 观察 AI 行为 → 填检查清单。
> **扮演**：非技术产品经理，只说业务话，不提 `data-action`/`IR`/`theme.css` 等技术词。

---

## S1 · 口述单页（回归）

**粘贴：**

```
我要做一个"供应商证照管理"页面，能按供应商名称和证照状态查询，列表显示供应商、证照名称、有效期、状态，能新增证照。
```

> 观看点：走老流程，不出现分诊/两道门/拆分任何字眼。

---

## S2 · 单文档做一页（回归）

**附上文件：** `绿星智审-审计监督模型_PRD_需求规格说明书_V1.md`（路径：`test-01/审计监督模型_2026年6月30日/审计监督模型/`）

**粘贴：**

```
帮我把"任务管理列表"这一页做出来。
```

> 观看点：不触发任何新机制，正常走步骤0→7。

---

## G1-1 · 要求像素级复刻（门一）

**不留文件，直接粘贴：**

```
我有个旧原型页面，帮我在新站点里做得一模一样，像素级的。
```

> 观看点：立即停下 → 给 A/B/C 三条路 → 大白话讲代价 → 不出现 CSS 术语。

**AI 停下后你回复：**

```
选B吧，尽量接近原来的样子。
```

> 观看点：立刻执行，不追问"你确定吗"。

---

## G1-2 · 要求换 UI 风格（门一第二信号）

**不留文件，直接粘贴：**

```
帮我把这套原型换成 Ant Design 的风格，不要现在这个绿星链通的样式。
```

> 观看点：识别信号 → 停下 → 说明"只能接近，正式换要另立项" → 给选项。

**AI 停下后你回复：**

```
那就尽量接近吧。
```

## G2-1 · 34个旧原型触发高风险（门二）

**附上文件：** PRD + 编码表 + 整个旧原型目录（34个html）
> 路径：`test-01/审计监督模型_2026年6月30日/审计监督模型/`

**粘贴：**

```
这是我们之前的需求文档、模块编码表和旧原型，你帮我重新做成新的原型站点。
```

> 观看点：先侦察再报告（不能先读文件）→ 必须说"34个页面文件"这个数字 → 四段式 → 给拆分建议 → 不拒绝。

## G2-2 · 用户说"全做"

```
别管了，全部做吧，一页页来就行。
```

> 观看点：不再劝 → 开始执行 → 交付说明有"已提示风险后执行"留痕。

## G2-3 · 单页不误触发

## G2-4 · 边界值：恰好15个文件

**附上文件：** 从旧原型目录中精确挑选 15 个 html 文件（不附 PRD 和编码表）

**粘贴：**

> 观看点：执行侦察但不输出"高风险"诊断（15 是中风险上限，不触发高风险）。

## G2-5 · 边界值：2类材料+6个文件

**粘贴：**

```
帮我做这几个页面。
```

> 观看点：识别到 2 类材料但不触发高风险诊断（需 ≥3 类），正常进入逐页流程。

## SPLIT-1 · 完整拆分流程

**同 G2-1 输入（PRD + 编码表 + 所有 html）。AI 输出诊断报告后粘贴：**

```
行，你先帮我把这些拆成独立的小提示词文件吧，我来确认。
```

> 观看点：若站点不存在先建骨架 → 产出 N 份文件到 prompts/ → 列出清单 → 等你勾选。

**AI 列出清单后粘贴：**

```
先做前3个：任务管理列表、任务创建、模型列表。
```

> 观看点：每页生成前重新读该页提示词文件 → 逐页生成、逐页确认。

---

## SPLIT-2 · 跳过拆分

**同 G2-1 输入，AI 输出诊断报告后粘贴：**

```
别拆了，你看着办，全部做吧。
```

> 观看点：提醒一句风险 → 确认后执行 → 不反复劝。

---

## HANDOFF-1 · 批次结束出交接文档

**用 SPLIT-1 会话，做完 3 页后粘贴：**

```
这批就先到这吧，剩下的后面再做。
```

> 观看点：主动产出交接文档 → 包含已完成清单/待办清单/下一步建议 → 问"要不要开新会话"。**记录交接文档路径，HANDOFF-2 要用。**

---

## HANDOFF-2 · 新会话读交接文档接续

**⚠️ 必须新开一个会话。只附上 HANDOFF-1 产出的交接文档，粘贴：**

```
上一批的进度在这份文档里，你接着做剩下的页面。
```

> 观看点：
> - 交接文档信息完整 → 直接开始做下一页，不重问需求
> - 交接文档关键字段为空 → AI 追问"站点路径？"是**合理行为**，不判失败（记录为交接文档质量问题）

## META-1 · 小偏好不啰嗦

**在任一用例中途穿插粘贴：**

```
示例数据给我放20条，新增按钮的字改成"登记"。
```

> 观看点：直接照做，不触发风险说教。

---

## META-2 · 风险只讲一次

**复用 G2-2 和 G1-1 的回合，不需单独开会话。观察：**

- G2-2 你说"全部做"后 → AI 若再次提醒风险 = 违规
- G1-1 你选 B 后 → AI 若追问"你真的要选B吗" = 违规

---

## META-3 · 脏活自己扛

**复用 SPLIT-1 和 HANDOFF-1 的回合，不需单独开会话。观察 AI 是否：**

- 主动提拆分建议（不需要你说"帮我分拆"）
- 主动更新交接文档（不需要你说"帮我建文档记录进度"）
- 主动问要不要开新会话（不需要你说"清一下上下文"）

## STYLE-1 · 新组件样式可用

**不留文件，直接粘贴：**

```
帮我做一个"风险驾驶舱"页面。顶部放4个统计卡片（待处理风险数、已处置风险数、超期风险数、同比趋势），每个卡片左边有个彩色图标块，hover浮起来。下面左边放趋势折线图（占三分之二宽度），右边放风险类型分布饼图。再下面放风险列表，有等级标签（高/中/低不同颜色），每行有操作按钮组。列表上方有状态页签（全部/待处理/已处置），页签上有数字徽章。
```

> 观看点：生成的页面源码中搜索这些类名，至少命中 3 个：
> `.page-header` `.title-badge` `.st-count` `.type-tag` `.stat-card.rich` `.level-tag` `.op-btns`
