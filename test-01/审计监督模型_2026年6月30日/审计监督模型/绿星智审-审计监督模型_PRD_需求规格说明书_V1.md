## 2 概述

### 2.1 目的

本文档旨在明确「绿星智审-审计监督模型」的产品定位、功能范围、业务流程和界面原型，为后续开发实现提供统一依据。本文档面向项目经理、开发团队、测试团队及相关干系人，确保各方对产品需求有一致的理解。

### 2.2 项目背景

国务院国资委先后下发1号文《关于推动中央企业加快财务数智化转型升级的指导意见》、2号文《关于加强中央企业穿透式监管的指导意见（试行）》和15号文《关于做好2026年中央企业内部控制体系建设与监督工作有关事项的通知》等系列文件，明确要求央企加快数字化智能化监管能力建设。中铝集团作为有色金属行业龙头企业，积极响应政策要求，依托「绿星链通」采购数字化平台，建设「绿星智审」审计监督模型系统。

当前采购业务面临五大痛点：

- 供应商准入管理粗放，关联关系识别困难
- 寻源过程透明度不足，围标串标行为难以发现
- 合同执行缺乏有效监督，履约风险频发
- 结算环节审核依赖人工，效率低且易遗漏
- 风险事件处置流程分散，缺乏端到端闭环管理

本项目旨在通过AI技术（规则模型+指标模型+大模型）实现采购全流程风险识别、合规审查、违规预警及闭环处置，提升采购监督的智能化水平。

### 2.3 适用范围

#### 2.3.1 涉及部门及单位

本系统面向中铝集团及下属各级经营单元的采购管理部门、审计监督部门、纪检监察部门、供应商管理部门及相关业务执行人员。

#### 2.3.2 覆盖业务范围

覆盖采购供应链全流程关键环节：供应商准入→采购计划→寻源（招标/询比价/单一来源）→合同签订→合同执行→结算付款→供应商评价。

#### 2.3.3 应用系统/功能范围

本系统作为「绿星链通」平台的监督子模块，功能范围包括：模型管理（规则模型/指标模型/综合模型）、风险事件管理、任务中心（含整改闭环）、风险驾驶舱及统计报表。

### 2.5 术语定义

| 术语 | 说明 |
|------|------|
| 风险模型 | 用于识别采购业务风险的规则、指标或算法的统称，分为规则模型、指标模型、综合模型三类 |
| 规则模型 | 基于预设条件（IF-THEN逻辑）的风险检测模型，运行在规则引擎之上，实现毫秒级高频风险检测 |
| 指标模型 | 基于数据聚合的统计/机器学习模型，用于发现统计规律异常 |
| 综合模型 | 基于大模型语义理解和知识图谱推理的复合模型 |
| 风险事件 | 模型识别出的风险记录，经确认后进入处置流程 |
| 监督任务 | 由风险事件触发的整改/核实任务，指派给责任人处理 |
| 穿透式监管 | 通过数据关联分析实现从集团到下属单位的风险穿透监控 |

## 3 系统说明

### 3.1 产品定位

「绿星智审」是中铝集团绿星链通平台的采购供应链审计监督子系统，以"数据为基、模型为核、闭环为要"为建设理念，利用AI技术实现采购全流程风险识别、合规审查、违规预警及闭环处置，构建覆盖"模型配置→风险识别→事件处置→整改跟踪→归档入库"的全链路监督体系。

### 3.2 业务流程

系统核心业务流程如下：

```
【模型配置】 → 【模型执行】 → 【风险识别】 → 【风险事件】 → 【任务创建】 → 【任务指派】 → 【任务处理】 → 【整改验收】 → 【归档入库】
```

**关键流程说明：**

- **模型层：** 通过规则模型（16个）进行毫秒级高频检测，通过指标模型（11个聚合模型）发现统计规律异常，通过综合模型（2个）进行深度语义理解和关联推理。
- **风险事件层：** 模型检测到的异常根据预设阈值产生风险事件，经过等级评定后进入处置流程。
- **任务层：** 风险事件触发监督任务创建，系统自动或手动指派责任单位/责任人进行整改核实。
- **处置层：** 责任人填报整改方案→执行整改→业务核实确认→跟踪验收→归档入库，形成完整闭环。

| 角色 | 职责描述 | 核心操作 |
|------|---------|---------|
| 系统管理员 | 系统配置、用户管理、角色权限分配、运行监控 | 用户维护、角色配置、模型发布/启停、执行监控 |
| 模型管理员 | 风险模型的配置、调试、发布和版本管理 | 规则配置、指标配置、模型测试、版本管理 |
| 风控专员 | 风险事件确认、等级评定、处置跟踪 | 风险事件查看/录入、等级调整、处置跟踪 |
| 业务责任人 | 接收并处理监督任务，填报整改方案 | 查看我的任务、整改方案填报、任务进度反馈 |
| 审计领导 | 风险总览、任务督办、统计分析和决策支持 | 驾驶舱查看、统计报表、超期督办、验收审核 |
| 供应商管理员 | 供应商风险监控和评估 | 供应商评分卡查看、评估结果查询 |

### 3.4 产品范围

本系统覆盖采购供应链监督的六大业务场景：

- **准入环节场景：** 供应商关联关系挖掘、准入资质合规性检查、黑名单拦截
- **寻源场景（含非招标）：** 围标串标识别（IP一致/文件雷同等）、评标专家异常行为分析、应招未招检测
- **合同场景：** 合同到期预警、超期未执行提醒、合同智能校验
- **结算场景：** 付款异常检测、发票合规校验、三单匹配核验
- **供应商评价场景：** 供应商评分卡、动态风险扫描、供应商分类分级
- **综合监督场景：** 采购业务健康度评估、合规规则知识检索

### 4.2 功能模块详述

#### 【P0 第一阶段 — 核心流程打通】

目标：完成从模型配置→模型执行→风险事件→任务创建→任务指派→任务处理的主流程串联，实现"风险可识别、事件可处置、任务可跟踪"的完整闭环。

##### 4.2.1 模型中心

| 子功能 | 优先级 | 说明 |
|--------|--------|------|
| 模型列表 | P0 | 按类型（规则模型/指标模型/综合模型）分类展示全部模型，支持搜索、筛选、启停操作。综合模型入口预留但置灰显示 |
| 规则配置管理 | P0 | 规则模型列表页，展示规则名称、状态、命中次数、最后执行时间等字段，支持查询、启用/停用、编辑操作 |
| 规则配置编辑 | P0 | 规则条件编辑器，支持配置规则名称、所属场景、触发条件（字段+运算符+阈值）、风险等级、处置方式等 |
| 规则命中日志 | P0 | 展示每条规则模型的执行命中记录，包括命中时间、触发数据、风险等级、是否生成事件等 |
| 指标管理+新增 | P0 | 指标定义管理，支持指标名称、计算口径、数据来源、阈值范围等配置 |
| 采购业务健康度 | P0 | 以仪表盘形式展示采购业务健康评分，支持编辑指标权重和阈值，查看评分详情 |
| 供应商评分卡 | P0 | 按多维度（质量/交付/价格/服务）对供应商进行评分，支持评分模型编辑 |
| 评估结果查询 | P0 | 查询指标模型的评估输出结果，支持按模型、时间、结果范围筛选 |
| 执行状态看板 | P0 | 模型执行状态监控面板，展示各模型运行状态（正常/异常/未执行）、最近执行时间、执行耗时 |
| 执行结果查询 | P0 | 查询模型执行历史记录，查看每次执行的输入参数和输出结果 |

> **新增功能：模型执行命中→风险事件自动生成（原型缺失，本次新增）**

**业务规则：** 模型执行命中后，系统根据命中规则的严重程度和置信度自动生成风险事件，或由风控专员在"执行结果查询"中点击"转事件"手动生成。自动生成阈值可在规则配置中设定。

**关键字段：** 风险事件编号（自动生成，规则：FX+日期+序号）、来源模型、命中规则、风险等级（高/中/低）、事件描述、涉及供应商、涉及金额、事件状态（待确认/已确认/处置中/已关闭）。

##### 4.2.3 任务中心

| 子功能 | 优先级 | 说明 |
|--------|--------|------|
| 任务创建 | P0 | 创建整改任务，支持从风险事件自动带入信息。关键字段：任务标题、关联风险事件、责任单位、责任人、整改期限、任务描述。新增「指派」功能 |
| 任务管理列表 | P0 | 全部任务列表，管理员视角。支持按状态、责任单位、期限等多维度筛选 |
| 任务详情 | P0 | 任务完整信息展示，包含关联风险事件跳转链接、处置进度时间线、整改方案、验收记录 |
| 我的任务 | P0 | 当前用户的待办/已办任务列表，支持查询和快速处理入口 |
| 整改方案填报 | P0 | 责任人填写整改措施、计划完成时间、所需资源等，支持上传附件 |
| 任务验收审核 | P0 | 验收人对整改结果进行审核，支持通过/驳回操作，驳回需填写驳回原因 |
| 业务核实确认 | P0 | （内嵌组件）核实风险实际情况，确认业务数据是否存在异常，支持"确认异常/误报" |
| 跟踪验收 | P0 | （内嵌组件）跟踪整改执行进度，有时间线组件展示处置全流程 |
| 归档入库 | P0 | （内嵌组件）验收通过后归档，完结处置流程。归档后任务不可再修改 |
| 超期督办 | P0 | 超期未完成任务列表展示，支持发送督办通知、记录督办结果 |

> **新增功能：任务指派功能（原型缺失，本次新增）**

**业务规则：** 任务创建时需指定"责任单位"和"责任人"。系统提供组织架构树和用户搜索，选择后任务自动出现在对应责任人的"我的任务"列表中。支持加签（多人协办）和转办功能。

## 5 非功能性需求

### 5.1 界面需求

- 系统采用统一UI风格，遵循「绿星链通」平台的设计规范
- 支持主流浏览器最新版本（Chrome/Edge/Firefox）
- 页面布局采用固定宽度+自适应内容区，最低分辨率支持1366×768
- 菜单采用多级树形结构，支持展开/折叠
- 表格列表支持分页、排序和列筛选
- 表单页面需提供必填项校验和错误提示

### 5.2 性能需求

- 页面加载时间 ≤ 3秒（首次）/ ≤ 1秒（缓存后）
- 列表查询响应时间 ≤ 3秒（10万条数据以内）
- 规则引擎单次命中检测 ≤ 500毫秒
- 支持并发用户数 ≥ 100

### 5.3 安全与权限

- 角色权限：按角色控制菜单可见性和操作权限（查看/新增/编辑/删除/审批）
- 数据权限：不同角色和用户只能查看其权限范围内的数据（按组织架构隔离）
- 敏感数据传输和存储需加密
- 用户操作需记录关键操作日志，支持追溯

### 5.4 运维需求

- 支持系统参数在线配置，无需重启服务
- 支持模型执行日志的周期性归档和清理
- 支持数据导入/导出（CSV/Excel格式）

### 5.5 接口需求

本系统需要与以下外部系统进行数据交互：

| 外部系统 | 接口方向 | 数据内容 | 对接状态 |
|---------|---------|---------|---------|
| 绿星链通平台 | 双向 | 采购业务数据、供应商数据、合同数据 | 同平台内部对接 |
| 坤安大模型 | 单向调用 | 模型推理请求与结果 | 待确定接口协议 |
| SAP系统 | 单向读取 | 财务结算数据 | 待对接 |
| 集团MDM主数据 | 单向读取 | 供应商主数据、组织架构 | 待对接 |

## 6 验收标准

### 6.1 功能验收标准

**P0阶段验收条件（核心流程）：**

- 当配置规则模型条件后，运行模型可产生命中日志
- 当模型命中后，可生成风险事件（自动或手动）
- 当风险事件确认后，可创建任务并指派给责任人
- 当任务指派后，责任人可在「我的任务」中看到任务
- 当责任人填报整改方案后，验收人可进行验收审核
- 当验收通过后，任务可归档入库，流程完结
- 当任务超期后，可在督办列表中查看

**P1阶段验收条件（驾驶舱）：**

- 驾驶舱统计数字与明细列表数据一致
- 统计报表支持多维度筛选和图表展示
- 首页工作台统计卡片点击后跳转至对应明细页面

### 6.2 非功能验收标准

- 性能：在约定硬件环境下，页面加载时间满足性能指标
- 兼容性：在Chrome/Edge浏览器上正常显示和操作
- 安全性：通过安全扫描，无高危漏洞
