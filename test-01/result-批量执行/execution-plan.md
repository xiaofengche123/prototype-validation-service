## 三、执行流程（AI 必须遵循）

### Step 0：启动检查
1. 读取本文件（`execution-plan.md`）确认参数
2. 读取 `handoff.md`（如有）确认当前进度
3. 检查 `prompts/` 目录是否存在拆分文件
   - 无 → 执行 Step 1（拆分）
   - 有 → 执行 Step 2（生成）

### Step 1：PRD 拆分（仅首次执行）
1. 读取 PRD 源文件
2. 按 34 页清单逐页提取：
   - 页面编码、页面名称、页面类型、菜单归属
   - 查询字段（名称、类型、选项值）
   - 表格列（列名、数据格式、是否可排序）
   - 操作按钮（位置：顶部/行内，行为：弹窗/跳转/提交）
   - 交互要求（跳转目标、弹窗内容、校验规则）
3. 每页输出到 `prompts/<编码>-<页面名>.md`，约 30 行
4. 标记 PRD 中缺失的信息为 `【需补位】`
5. 生成本文件"四、批次划分"，等 PM 确认

### Step 2：逐批生成（循环执行）
每批标准动作：
1. **读取 handoff.md** 确认已完成/待办
2. **读取本批 prompts/*.md**（3-5 份）
3. **逐页生成**：
   - 按 `prototype-site` 规则生成页面
   - 更新 `menu.config.js`
   - 记录补位清单
4. **更新 progress.md**（实时记录，见下方格式）
5. **更新 handoff.md**（交接文档，见下方格式）
6. **问 PM**：下一批做哪几页 / 是否重开会话

## 七、风险预案

| 场景 | 处理方式 |
|------|---------|
| 拆分时 PRD 信息不足 | 标记 `【需补位】`，继续拆分，不阻塞 |
| 生成时 prompts 缺失字段 | 按常见做法补位，记录到 progress.md |
| 会话中断 | 新会话读取 handoff.md + 下批 prompts，自动接续 |
| PM 要求改批次 | 更新本文件"四、批次划分"，同步更新 handoff.md |
| 内嵌组件页型不确定 | 默认按弹窗处理，标记 `【需 PM 确认】` |
