# Codex 兼容改造交接说明（2026-07-13）

## 1. 改造目标

本次改造让同事从 `C:\work-project\psm-project\psm-ui\kimi-skill-project` 启动 Codex 时，可以发现并使用以下 4 个 Skill，同时保留 Kimi 现有使用方式：

- `prototype-guide`
- `prototype-site`
- `prototype-to-vue3`
- `prototype-vue3-framework`

改造原则是“一套业务规则、多个工具入口”：共享规则继续只维护在 `skills/`，Codex 入口只负责发现和转发，避免 Kimi 与 Codex 各自复制一份规则后逐渐不一致。

## 2. 兼容结构

### 2.1 唯一规则源

4 个 Skill 的完整业务规则、参考资料、脚本和资源仍位于：

```text
skills/<skill-name>/
```

Kimi 后续仍应读取这里的主 `SKILL.md`。修改公共业务规则时，也只修改这里。

### 2.2 Codex 发现入口

新增以下轻量入口：

```text
.agents/skills/<skill-name>/SKILL.md
.agents/skills/<skill-name>/agents/openai.yaml
```

入口 `SKILL.md` 不复制业务规则，只要求 Codex 完整读取 `../../../skills/<skill-name>/SKILL.md`，并以主 Skill 所在目录解析其相对引用。

没有采用符号链接，原因是当前 Windows 项目的 `core.symlinks=false`，符号链接在不同同事的 Git/Windows 权限配置下不稳定。轻量入口是普通文本文件，更适合作为团队项目文件分发。

### 2.3 项目级路由

根目录 `AGENTS.md` 已改为 Kimi/Codex/Claude 共用的简洁路由说明，主要负责：

- 明确 `skills/` 是唯一规则源；
- 区分 4 个 Skill 的职责边界；
- 说明 Codex 使用 `$skill-name` 显式调用；
- 区分生产 `psm-ui` 的 Vue2 技术栈与隔离 Vue3 原型框架；
- 说明从 `prototype-state.json`、`prototype-handoff.md`、`prototype-change-log.md` 续跑。

`.kimi-code/AGENTS.md` 和 `CLAUDE.md` 本次未主动重写，避免在尚未由对应工具复核前破坏其原有加载习惯。

## 3. 已完成的改造

### 3.1 Codex 元数据

- 为 4 个 Codex 入口增加 `agents/openai.yaml`；
- 为共享 `prototype-guide` 补充 `agents/openai.yaml`；
- 统一 4 个共享 Skill 的 `default_prompt`，都显式包含 `$skill-name`；
- 校正 `short_description` 长度到 Codex 推荐的 25–64 字符范围；
- 收紧 4 个 Skill 的触发描述，减少 `prototype-guide` 与 `prototype-site` 互相抢任务。

### 3.2 职责边界

- 原始需求、PRD、截图、旧 HTML：由 `prototype-guide` 负责访谈、拆页和确认；
- 已确认的页面规格：由 `prototype-site` 直接生成或更新静态页；
- 隔离 Vue3 演示壳：由 `prototype-vue3-framework` 创建或修复；
- 已注册静态原型转 Vue3：由 `prototype-to-vue3` 负责；
- 4 个 Skill 均不应直接改造生产 `psm-ui` 的 Vue2 页面。

### 3.3 公共规则修正

- 统一“内嵌组件不是独立页”的规则：命中 iframe 信号、“内嵌-”文件名前缀或组件型结构时，整合进宿主页，不注册菜单；
- 修正模板中跨 Skill 的相对路径；
- 修正旧编码兼容规则：新编码保持 6 位，旧编码允许 `^[A-Z]{2,6}[0-9]{2}$`，Vue 路径按末尾两位数字之前的字母部分拆分；
- 清理不合规编码示例和模板属性拼写错误；
- 统一批次建议：文档输入 3–5 页/批，旧 HTML 输入 2–3 页/批，复杂页面单页处理，首次试跑 1 页；
- 修正对不存在的 `prototype-site-change-log.md` 的引用。

### 3.4 样式和校验闭环

- 静态原型页面 head 只引用 `assets/app.css`；
- `app.css` 必须在首部且仅一次导入 `theme.css`；
- 禁止页面 `<style>` 块和任意内联布局样式，只保留表格/数据驱动场景的有限白名单；
- `theme.css` 只允许皮肤变量和视觉覆盖，不允许布局属性；
- `check-site.js` 新增主题导入检查、内联样式检查和内嵌参考页识别；
- 修复 `check-site.js` 的 C6 成功状态统计；
- `check-prompts.js` 支持指定提示词文件以及 `--html-only`，避免续跑时被历史批次无关文件阻塞。
